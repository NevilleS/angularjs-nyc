The problem...

<span class="big">{{ points }}</span>
<span>points</span>
<a class="button" href="#"
  ng-click="++points">
  Add points
</a>

"++points"

is an invalid Angular expression.

Angular expressions look like Javascript, and act like Javascript, but they aren't Javascript. They are their own language, implemented inside Angular by $parse.

ng-click="foo()"
ng-show="foo != bar"
Today is {{timestamp | date}}

Angular's $parse service takes strings like:

"data"

and converts them to executable functions like:

function (s, l) {
  if(s == null) return undefined;
  s=((l&&l.hasOwnProperty("data"))?l:s).data;
  return s;
}

How does it work?

(https://github.com/angular/angular.js/blob/master/src/ng/parse.js)

Is it slow?

Yes, but it's OK. Usually only called when compiling/linking templates, not during digest cycles (see ng-model, ng-bind, ng-click, etc.)

The solution...

<span class="big">{{ points }}</span>
<span>points</span>
<a class="button" href="#"
  ng-click="points = points + 1">
  Add points
</a>
